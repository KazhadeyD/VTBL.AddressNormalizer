# Каталог: VTBL.AddressNormalizer.WebApi

## Назначение
ASP.NET Core 5.0 HTTP-хост: endpoints, middleware, NLog, options; ядро через `AddAddressNormalizer()` + `IAddressNormalizationService`.

## Файлы

### Program.cs
**Классы:**
- `Program` — точка входа; NLog (`ClearProviders` + `UseNLog`), `UseStartup<Startup>`

### Startup.cs
**Классы:**
- `Startup` — DI: `BatchOptions`, Controllers + `ApiExceptionFilter`, Swagger, `AddAddressNormalizer()`, `IAddressNormalizationService` → `AddressNormalizationService`; pipeline: `CorrelationIdMiddleware` до routing

### nlog.config
Файловый конфиг NLog: console + file; layout с `${mdlc:item=CorrelationId}`; Microsoft.* → Warn+, остальное → Info+

### appsettings.json
Секция `Batch:MaxItems` (default 100)

## Подкаталоги
- `Controllers/` — Health + v1: Normalize / Unit / Address (только через IAddressNormalizationService)
- `Filters/` — MVC filters
- `Middleware/` — HTTP middleware
- `Models/` — DTO запросов/ответов
- `Mapping/` — IndoorValueMapper → IndoorValueDto
- `Services/` — IAddressNormalizationService + AddressNormalizationService (DI)
- `Options/` — typed configuration

## Зависимости
- `VTBL.AddressNormalizer.Abstractions`
- `VTBL.AddressNormalizer.Infrastructure`
- Packages: `NLog.Web.AspNetCore`, `Swashbuckle.AspNetCore`
