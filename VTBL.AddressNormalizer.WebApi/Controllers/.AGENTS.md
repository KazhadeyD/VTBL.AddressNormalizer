# Каталог: VTBL.AddressNormalizer.WebApi/Controllers

## Назначение
Тонкие HTTP controllers: binding DTO, маппинг HTTP-кодов, вызов только `IAddressNormalizationService` (без Factory).

## Файлы

### HealthController.cs
**Классы:**
- `HealthController` — `GET /health` → `{ "status": "Healthy" }` (без сервиса нормализации)

### NormalizeController.cs
**Классы:**
- `NormalizeController` — route `api/v1/normalize`
  - `POST /` → `NormalizeFull` → `NormalizeResponse` / `400 { error }`
  - `POST /batch` → `NormalizeBatch` + `BatchOptions.MaxItems` → `200 { items }` / `400|500 { error }`

### UnitController.cs
**Классы:**
- `UnitController` — route `api/v1/unit`
  - `POST /normalize` → `NormalizeUnit` → `UnitNormalizeResponse` / `400 { error }`

### AddressController.cs
**Классы:**
- `AddressController` — route `api/v1/address`
  - `POST /extract` → `ExtractOutdoor` → `ExtractResponse` / `400 { error }`
  - `POST /canonicalize` → `Canonicalize` → `CanonicalizeResponse` / `400 { error }`

## Зависимости
- `Services/IAddressNormalizationService`
- `Options/BatchOptions` (только NormalizeController)
- `Models/*` (SourceRequest, response DTO, ErrorResponse)
