# Каталог: VTBL.AddressNormalizer.WebApi/Services

## Назначение
Application-оркестрация бизнес-endpoint’ов нормализации. Контроллеры вызывают только этот слой. Зависимости ядра — через DI (`AddAddressNormalizer`).

## Файлы

### IAddressNormalizationService.cs
**Интерфейс:**
- `NormalizeFull(source)` → `NormalizeValueDto` (validation → `ArgumentException`)
- `NormalizeUnit(source)` → `UnitNormalizeResult`
- `ExtractOutdoor(source)` → `string`
- `Canonicalize(source)` → `string`
- `NormalizeBatch(sources, maxItems)` → `BatchOutcome`

### AddressNormalizationService.cs
**Классы:**
- `AddressNormalizationService` — оркестрация через ctor-injected ядро
  - ctor: `ILogger`, `IBuildingLocationExtractor`, `IBuildingAddressCanonicalizer`, `IBuildingUnitNormalizer`, `ICanonicalHash`
  - `NormalizeFull` — `ExtractSplit` → `ToCanonical(outdoor)` → `ComputeSha256` → `Normalize(indoor)` → `IndoorValueMapper.ToIndoorValueDto`; `fias_id = null`
  - `NormalizeUnit` — unit normalizer + mapper; без `category`
  - `ExtractOutdoor` — `ExtractSplit().Outdoor` (indoor клиенту не отдаётся)
  - `Canonicalize` — `ToCanonical(source)` **без** extract; hash не считается
  - `NormalizeBatch` — request-level (null/empty/>MaxItems → `RequestInvalid`); per-item: whitespace → validation error; иначе `NormalizeFullCore` в try/catch; ≥1 ok → `PartialOrSuccess`; all-fail → `AllFailValidation` / `AllFailException` / `AllFailMixed`
  - `NormalizeFullCore` (protected virtual) — seam для all-fail exception без мока ядра
  - `ILogger`: Information на старт; Warning на validation / item failure

### BatchOutcome.cs
**Типы:**
- `BatchOutcomeKind` — `PartialOrSuccess`, `AllFailValidation`, `AllFailException`, `AllFailMixed`, `RequestInvalid`
- `BatchOutcome` — `Kind`, `Items`, `ErrorMessage`
- `UnitNormalizeResult` — `Source`, `IndoorValue`, `Canonical`, `Hash`
