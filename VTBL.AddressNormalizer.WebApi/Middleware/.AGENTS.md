# Каталог: VTBL.AddressNormalizer.WebApi/Middleware

## Назначение
HTTP middleware pipeline.

## Файлы

### CorrelationIdResolver.cs
**Классы:**
- `CorrelationIdResolver` — чистая функция выбора Correlation Id
  - константы: `CorrelationIdHeaderName`, `RequestIdHeaderName`, `MdlcKey`
  - `Resolve(correlationIdHeader, requestIdHeader)` — Correlation → Request → GUID `"D"`; whitespace = отсутствует

### CorrelationIdMiddleware.cs
**Классы:**
- `CorrelationIdMiddleware` — выбор Id + NLog MDLC `CorrelationId` + response header `X-Correlation-Id`
  - `InvokeAsync(HttpContext)` — MDLC до `_next`, echo через `OnStarting`

### RequestLoggingMiddleware.cs
**Классы:**
- `RequestLoggingMiddleware` — после ответа: `HTTP {Method} {Path} → {StatusCode} за {ElapsedMs} ms`
  - 2xx → Information, 4xx → Warning, 5xx → Error
  - skip: `/health`, `/swagger`
  - должен идти **после** CorrelationIdMiddleware (Correlation Id в layout NLog)

## Зависимости
- NLog (`ScopeContext` / MDLC layout `${mdlc:item=CorrelationId}`)
- `Microsoft.Extensions.Logging`
